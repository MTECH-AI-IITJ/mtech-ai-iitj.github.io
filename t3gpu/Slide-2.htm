🎯 Slide 3 Programming Models (SIMD vs SIMT)

🧠 Easy Explanation

This slide introduces two key GPU programming models SIMD and SIMT, which define how instructions are executed on multiple data elements.

🟢 SIMD Single Instruction, Multiple Data

  • One instruction operates on multiple data items simultaneously.
  • Common in CPUs and vector processors.

  • Threads have individual registers and control flow, so they can diverge slightly (unlike pure SIMD).
  • Its like many independent workers following the same task description.

🎯 Slide 4 GPU Microarchitecture (Warps and SMs)

🧠 Easy Explanation

This slide explains the execution unit structure inside a GPU specifically how threads, warps, and streaming multiprocessors (SMs) interact.

⚙️ Warp

  • A warp = 32 threads executing the same instruction simultaneously.

  • If a warp executes an instruction, all its 32 threads execute it at once (possibly on different data).

Checking the GPU's Work Capacity (Occupancy)

Slide 14: Latency, Throughput, and Warps (Sources 37-38)

The Formula for Max Performance

The GPU uses warps to hide problems. If one warp has to wait for data (a problem called Latency), the processor quickly switches to another warp to keep working (this is high Throughput). The number of warps you need to perfectly hide all the waiting time is found by this simple formula27:

$$\text{Number of Required Warps} = \text{Latency} \times \text{Throughput}$$

    • Answer: $15 \text{ Cycles} \times 4 \text{ Operations/Cycle} = 60 \text{ Required Warps}$.

4. SIMT vs SIMD

  • SIMT: One instruction, many threads (each thread may work on different data).
  • SIMD: One instruction, many data (all threads do the same thing on different data).

5. Warp

  • A warp is a group of 32 threads.
  • All threads in a warp do the same thing at the same time.
  • Warps are the basic unit of execution in SMs.

6. Scheduling Decisions

  • Example: 1616 block = 256 threads.
  • SM can handle 1024 threads → 4 blocks per SM.
  • 256 threads/block → 8 warps/block → 32 warps total.
  • 32 warps/SM = 100% occupancy (fully used).

7. Warp Formula

  • Required Warps = Latency Throughput
    • Fermi: 20 32 = 640
    • Kepler: 20 192 = 3840

8. Fermi Architecture

  • SMs have:
    • Load/Store Units: Move data.
    • Special Function Units (SFUs): Do math like sin, cos, sqrt.
    • Register Files: Store temporary data.
    • Caches: Speed up memory access.

9. Memory Hierarchy

  • Global Memory: Big, slow.
  • Shared Memory: Fast, shared within block.
  • Local Memory: Private to thread.
  • Texture Memory: For images, read-only, fast access.

10. Workload Execution

  1. CPU starts the job.
  2. CPU sends data to GPU.
  3. GPU runs kernels (functions).
  4. Threads compute in blocks.
  5. Results go back to CPU.

11. Simple Processing Flow

  1. Copy data from CPU to GPU.
  2. Run GPU program.
  3. Copy results back to CPU.

12. Data Flow

  • Data moves from CPU → GPU → back to CPU.
  • Threads access memory through hierarchy.
